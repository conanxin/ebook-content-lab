## letters_brief.json Draft Schema
Top-level fields:
- project
- source_type
- status
- letter_count
- section_range
- source_artifacts
- letters
Each letter item:
- letter_id
- section_id
- order
- title
- char_count
- paragraph_count
- chunk_count
- chunk_ids
- brief
- places
- themes
- evidence_refs
- review_status
Evidence refs must be short and safe:
- no full section text
- no long paragraphs
- no quote over 80 characters
- no private absolute paths
## Builder Rules
build_letters_brief.py should:
1. Accept --project.
2. Use ProjectPaths from scripts/lib/project_paths.py.
3. Read book_sections.jsonl and book_chunks.jsonl.
4. Select only body letters, expected sec-006 through sec-030.
5. Create one letter record per body section.
6. Use title / counts / chunk ids from intake artifacts.
7. Generate conservative placeholder briefs from metadata and titles only if safe.
8. Write working/letters_brief.json.
9. Write reports/letters_brief_report_v0.7_a2.md.
## Checker Rules
check_letters_brief.py should:
1. Accept --project and --version.
2. Verify letters_brief.json exists.
3. Verify letter_count == 25.
4. Verify section coverage is exactly sec-006 through sec-030.
5. Verify no sec-001 to sec-005 or sec-031 appears as a body letter.
6. Verify every letter has required fields.
7. Verify evidence refs are short and safe.
8. Verify no private text or long excerpts leak.
9. Write reports/letters_brief_validation_v0.7_a2.md.
## Non-goals

- create public JSON
- modify web mirror data
- modify page code
- create manual review tasks
- change project status
