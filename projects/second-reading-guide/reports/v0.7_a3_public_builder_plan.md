# v0.7-A3 Public Builder Plan

## Goal

A3 converts the `second-reading-guide` project from placeholder public JSON files to conservative structured public reading-guide JSON files.

Current baseline:

- Project status remains `draft`.
- Source type is `epub`.
- A2 `working/letters_brief.json` exists and contains 25 body letters.
- Existing public JSON files were placeholder shells.
- A3 writes public JSON and web mirror JSON, but does not promote status.

## Strict Boundaries

A3 must not:

- modify `projects/second-reading-guide/private/`
- publish EPUB text
- publish `book.md`
- publish `book_sections.jsonl`
- publish `book_chunks.jsonl`
- publish absolute private paths
- modify page code
- promote status
- tag or release

## Inputs

A3 may read:

- `projects/second-reading-guide/working/letters_brief.json`
- `projects/second-reading-guide/working/book_identity.json`
- `projects/second-reading-guide/working/book_identity_source.json`

Unsafe fields must not be copied into public output:

- absolute `epub_path`
- `private/source/book.epub`
- `book.md`
- `book_sections.jsonl`
- `book_chunks.jsonl`
- `完整正文`
- `原始正文`
- `章节正文`

## Outputs

A3 writes 5 project public JSON files:

- `public/book_overview.json`
- `public/chapter_reading_cards.json`
- `public/key_concepts.json`
- `public/quote_index.json`
- `public/reading_questions.json`

A3 mirrors the same 5 files to:

- `web/public/projects/second-reading-guide/`

A3 also writes:

- `reports/reading_guide_public_report_v0.7_a3.md`
- `reports/reading_guide_public_validation_v0.7_a3.md`

## Schema Decision

A3 upgrades schema from:

- `reading-guide.v0.1`

to:

- `reading-guide.v0.2`

Reason:

- v0.1 files were placeholder shells.
- v0.2 includes A2-derived structural fields such as `places`, `themes`, `chunk_count`, and structural evidence refs.
- v0.2 still avoids source text and long excerpts.

## Status Decision

A3 keeps status as:

- `draft`

A3 does not promote to `content-draft`.

Status promotion is deferred to a future `promote_status.py` phase.

## Public File Rules

### book_overview.json

Filled from safe metadata and `letters_brief` statistics:

- `schema_version`
- `status`
- `book`
- `one_sentence_summary`
- `reading_purpose`
- `structure_overview`
- `how_to_use`
- `limitations`
- `evidence_refs`

### chapter_reading_cards.json

Generates 25 cards, one per body letter:

- `chapter_id`
- `letter_id`
- `section_id`
- `order`
- `title`
- `summary`
- `places`
- `themes`
- `char_count`
- `paragraph_count`
- `chunk_count`
- `evidence_refs`
- `review_status`

### key_concepts.json

Generates minimal metadata-derived concepts by aggregating A2 `themes`.

These concepts are structural drafts, not deep interpretation.

### quote_index.json

Uses `structural_no_quote`.

Rules:

- real quotes are not generated in A3
- `quote` fields remain empty
- entries preserve future manual review slots only

### reading_questions.json

Generates lightweight structural questions from title / places / themes.

Questions are generic draft prompts, not final interpretive claims.
