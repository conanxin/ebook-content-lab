# v0.7-A4 Manual Review Workflow Plan

## Goal

v0.7-A4 adds a manual review workflow for the `second-reading-guide` project.

The goal is to generate a review task set that lets a human verify the public reading-guide draft before any status promotion.

## Inputs

A4 reads only structural working and public artifacts:

- `projects/second-reading-guide/working/letters_brief.json`
- `projects/second-reading-guide/public/book_overview.json`
- `projects/second-reading-guide/public/chapter_reading_cards.json`
- `projects/second-reading-guide/public/key_concepts.json`
- `projects/second-reading-guide/public/quote_index.json`
- `projects/second-reading-guide/public/reading_questions.json`

A4 does not read private source material or book body text.

## Outputs

A4 writes:

- `projects/second-reading-guide/reports/manual_review_tasks_v0.7_a4.csv`
- `projects/second-reading-guide/reports/manual_review_summary_v0.7_a4.json`
- `projects/second-reading-guide/reports/manual_review_dashboard_v0.7_a4.md`
- `projects/second-reading-guide/reports/manual_review_validation_v0.7_a4.md`
- `projects/second-reading-guide/reports/v0.7_a4_manual_review_plan.md`

## CSV Single Source Of Truth

The CSV file is the single source of truth for manual review.

The summary JSON and dashboard Markdown are generated from the CSV task list. Human review results should be filled into the CSV later, not inferred from generated summaries.

## Priority Definitions

- `P0`: Must review before promotion because the issue could affect public boundary, schema, or reader trust.
- `P1`: Should review before promotion because it affects content quality and coverage.
- `P2`: Optional refinement for later wording, grouping, quote workflow, and question quality.

## Category Definitions

Core P0 categories:

- `public_boundary`
- `book_overview`
- `chapter_card_sample`
- `quote_policy`
- `schema_status`
- `web_mirror`

P1 content coverage categories:

- `chapter_card`
- `key_concept`
- `reading_question`
- `quote_structural_entry`

P2 refinement categories:

- `wording_polish`
- `concept_grouping_refinement`
- `future_quote_replacement`
- `question_quality_enhancement`

## Why A4 Does Not Promote Status

A4 creates review tasks only. It does not perform human judgment and does not mark the content as reviewed.

Project status remains `draft` until manual review results are filled and checked.

## Suggested v0.7-A5 Promotion Conditions

A future `promote_status.py` phase should require:

- all P0 tasks have explicit human results
- no P0 blocker remains open
- public files still pass validation
- project-level sanity checks still pass
- release boundary checks still pass

Only after those checks should the project move beyond `draft`.
