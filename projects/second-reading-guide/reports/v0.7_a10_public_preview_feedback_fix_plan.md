# v0.7-A10 Public Preview Feedback Fix Plan

## User Feedback Summary

The public preview was reachable, but the first pass still felt like a technical data page.

Main feedback:

- Project naming should match the first project style and use the actual book title.
- The visible reading content was too thin.
- The page should feel like a correspondence / envelope reading experience.
- Reading questions need answer hints.
- More improvements should be implemented directly from the preview page feedback.

## A10 Goal

A10 is a targeted public preview improvement pass for `second-reading-guide`.

It improves naming, content density, reading question answers, and visual design while keeping the project in draft public preview.

## Naming Fix

The public title becomes:

`《旅行人信札》阅读导览`

The subtitle becomes:

`25 封旅行书信的路线、地点与主题导读`

The book title becomes:

`《旅行人信札》`

## Content Enhancement

Each of the 25 chapter cards receives additional public-preview guide fields:

- `letter_summary`
- `route_note`
- `reading_focus`
- `theme_note`
- `review_notice`

These fields are generated from title, place, theme, and structure metadata. They are not final interpretation.

## Reading Question Answers

Each of the 26 reading questions receives an `answer_hint`.

The answer hints are guide prompts, not standard answers. They are based on title, place, theme, and structure metadata, and remain pending manual review.

## Envelope UI

The reading page is redesigned around letter / envelope cards:

- numbered letter mark
- date stamp
- route label
- folded flap effect
- expandable reading clues
- mobile-friendly card layout

## Copyright and Source Boundary

A10 publishes only original guide summaries, structural metadata, and review-pending prompts.

It does not publish source book material, long excerpts, local source paths, or source-layer files.

## No Status Promotion

A10 does not mark the project reviewed, final, or publish-ready.

The public state remains:

- `status`: `draft`
- `release_phase`: `public-preview`
- `review_status`: `manual-review-pending`

Manual review remains incomplete, so promotion preflight must still block reviewed-draft promotion.

## A11 Suggestion

After users inspect the improved public preview, A11 should handle concrete feedback items one by one: wording edits, card ordering, concept grouping, answer-hint refinements, and manual-review task updates.
