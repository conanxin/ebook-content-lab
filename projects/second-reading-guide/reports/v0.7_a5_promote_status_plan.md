# v0.7-A5 Promote Status Preflight Plan

## Goal

v0.7-A5 adds a safe status promotion preflight for the `second-reading-guide` project.

The tool defines when a reading-guide project may move from `draft` to a later reviewed state, but this phase must not actually promote the project.

## Why A5 Does Not Promote Status

Manual review has not been completed. The A4 task CSV currently has blank human review results.

The correct A5 result is therefore:

- promotion readiness: false
- blocking reason: manual review incomplete
- applied: false
- public status remains `draft`
- web mirror status remains `draft`

A5 reads:

- `projects/second-reading-guide/reports/manual_review_tasks_v0.7_a4.csv`
- `projects/second-reading-guide/reports/manual_review_summary_v0.7_a4.json`

## Readiness Rules

Promotion is allowed only when:

- current public status is `draft`
- all public files use `reading-guide.v0.2`
- all five public files exist
- all five web mirror files exist and match the public files
- manual review CSV and summary exist
- CSV row count matches summary total
- all P0 and P1 tasks have human results
- all P0 and P1 tasks are `pass`
- P2 tasks are either `pass` or `deferred`
- every deferred P2 task has notes
- no task is `needs_fix` or `blocked`
- no unsafe source-body markers or private paths are present

## Allowed Manual Result Values

- blank: not reviewed
- `pass`
- `needs_fix`
- `blocked`
- `deferred`

Blank, `needs_fix`, `blocked`, and illegal values all block promotion.

## Dry-Run And Apply Safety

`promote_status.py` defaults to preflight behavior.

Public JSON and web mirror JSON can be changed only if all checks pass and `--apply` is used with an exact confirmation string:

`PROMOTE second-reading-guide FROM draft TO <target-status>`

A5 runs only dry-run mode.
