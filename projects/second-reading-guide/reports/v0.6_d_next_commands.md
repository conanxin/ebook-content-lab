## C. 如果 EPUB 解析失败

按以下顺序排查：

### C.1 疑似 DRM / 加密

- 症状：解压后 `META-INF/` 包含 `rights.xml`，或 OPF manifest 标记 `encrypted=true`，或抽出的文本是空白/乱码
- 处理：
  1. 在 `book_identity_report.md` 顶部明确写 `DRM suspected — DO NOT PROCEED`
  2. **不**抽取正文
  3. 在 `projects/second-reading-guide/review_pack/notes.md` 记录需要未加密版本
  4. 整体状态保留为 `needs_manual_review=true`

### C.2 EPUB 损坏

- 症状：zip 解压失败 / mimetype 内容不是 `application/epub+zip` / `META-INF/container.xml` 缺失
- 处理：
  1. 用 `unzip -t book.epub` 检查 zip 完整性
  2. 用其他 EPUB 阅读器（Calibre / Sigil）打开验证
  3. 必要时重新下载或重新生成 EPUB
  4. 在 `book_identity_report.md` 记录损坏位置

### C.3 图片型 EPUB

- 症状：spine 主要是图片文件（.jpg/.png），XHTML 文本极短
- 处理：
  1. 标记 `image-first=true`
  2. **不**进入 reading-guide 草稿生成
  3. 整体状态保留为 `needs_manual_review=true`，并标注 `requires_ocr=true`
  4. 当前流水线不做 OCR；如果未来要做，需要先扩展 `extract_epub.py`

### C.4 container.xml / OPF / nav 缺失

- 症状：脚本无法定位 OPF / 章节列表为空
- 处理：
  1. 检查 `META-INF/container.xml` 是否引用了正确的 OPF
  2. 检查 OPF 是否引用了 nav.xhtml 或 ncx
  3. 用 epubcheck（如果可用）做静态校验：`epubcheck book.epub`
  4. 在 `book_identity_report.md` 记录缺失的具体文件

## D. 识别成功之后

进入 reading-guide 公开数据生成（**v0.6-D1+**）。每一步都会**先**生成 `projects/second-reading-guide/public/<name>.json`，**再**镜像到 `web/public/projects/second-reading-guide/<name>.json`。

### D.1 生成 `book_overview.json`

由人工 + 抽取数据共同生成（建议先放草稿，再迭代）。字段要点见 `book_intake_checklist_v0.6.md` §7。

### D.2 生成 `chapter_reading_cards.json`

按 `book_identity_report.md` 里的章节列表，逐章写卡片：

- 标题
- 导读（1–2 段）
- 关键论点（3–5 条）
- `evidence_refs`（页码 / 章节锚点 + 短摘）

### D.3 生成 `key_concepts.json`

从章节正文抽取候选概念，人工筛掉误报。字段：

- 概念名
- 解释
- 出现章节列表
- 相关引文 ID

### D.4 生成 `quote_index.json`

每条引文：短摘（≤120 字符）+ 章节 / 页码 + 用途。**严守公开边界**（见 checklist §8）。

### D.5 生成 `reading_questions.json`

按章节提出阅读问题，对应章节 + 参考答案方向。

### D.6 校验流水线

每一步生成后跑：

```bash
python scripts/check_reading_guide_project.py --project projects/second-reading-guide
python scripts/check_public_release.py
cd web && npm run build
```

任何一步失败，先回看 `reading_guide_validation_report.md` 与 `public_release_report.md` 的 Errors / Warnings 段。

### D.7 上线

- commit（精确 add，不使用 `git add .`）
- push `origin main`
- 监控 `gh run watch <RUN_ID> --exit-status`
- 线上验收：`/#/projects/second-reading-guide` 的 5 个模块是否填充内容
