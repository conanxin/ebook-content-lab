# v0.7-A6 Manual Review Completion Helper Plan

## Goal

v0.7-A6 adds a safe helper for recording manual review results one task at a time.

This phase also adds a progress checker so the project can report review completion without changing public reading-guide status.

## Why A6 Does Not Auto-Fill Results

Manual review must be performed by a human. A script must not mark tasks as passed just because the project generated valid structured data.

A6 therefore runs only a dry-run update example and leaves the A4 CSV unchanged.

## Inputs

A6 reads:

- `projects/second-reading-guide/reports/manual_review_tasks_v0.7_a4.csv`
- `projects/second-reading-guide/reports/manual_review_summary_v0.7_a4.json`
- `projects/second-reading-guide/reports/promote_status_preflight_v0.7_a5.md`

A6 does not read private book body material.

## Outputs

A6 writes:

- `projects/second-reading-guide/reports/manual_review_progress_v0.7_a6.md`
- `projects/second-reading-guide/reports/manual_review_progress_validation_v0.7_a6.md`
- `projects/second-reading-guide/reports/manual_review_completion_usage_v0.7_a6.md`
- `projects/second-reading-guide/reports/v0.7_a6_manual_review_completion_helper_plan.md`

## Update Helper Safety

`update_manual_review_result.py` defaults to dry-run behavior.

To write CSV changes, it requires:

- `--apply`
- an exact confirmation string: `UPDATE MANUAL REVIEW TASK <resolved-task-id>`
- a legal result value
- notes for `deferred`, `needs_fix`, and `blocked`

The helper does not support blanket automatic pass updates.

## Progress Checker Rules

`check_manual_review_progress.py` reports:

- total task count
- blank result count
- pass count
- needs-fix count
- blocked count
- deferred count
- P0/P1/P2 completion
- blockers
- readiness for promotion

Current expected state:

- total tasks: 95
- blank results: 95
- readiness for promotion: false
- reason: manual review incomplete

## Relationship To A5

A5 promotion preflight remains blocked until manual review results are filled and pass readiness rules.

A6 provides the controlled update mechanism and progress report needed before rerunning the A5 preflight.

## Suggested A7

A7 should perform real manual review updates using the helper, or add a batch-import flow for human-reviewed CSV edits with the same validation rules.
