# v0.7-A1 ProjectPaths Path Resolver

## Summary

v0.7-A1 adds a minimal shared path resolver for ebook subprojects.

The current `second-reading-guide` state remains:

- status: `draft`
- source type: `epub`
- pipeline baseline: `intake_ready`
- reviewed-draft: not required
- manual review summary: not required
- public content counts such as `26/10/68/32`: not required

## Added Path Fields

`ProjectPaths` covers these key path groups:

- Repository and project:
  - `repo_root`
  - `project_dir`
  - `slug`
  - `project_json`
- Private input:
  - `private_dir`
  - `source_dir`
  - `epub_path`
- Working artifacts:
  - `working_dir`
  - `book_identity_json`
  - `book_identity_source_json`
- Reports:
  - `reports_dir`
  - `epub_extraction_report`
  - `book_identity_report`
- Project public data:
  - `public_dir`
  - `book_overview_json`
  - `chapter_reading_cards_json`
  - `key_concepts_json`
  - `quote_index_json`
  - `reading_questions_json`
- Web public mirror:
  - `web_public_dir`
  - `web_project_dir`
  - `web_index_json`
  - `web_book_overview_json`
  - `web_chapter_reading_cards_json`
  - `web_key_concepts_json`
  - `web_quote_index_json`
  - `web_reading_questions_json`

Helper methods:

- `report_path(name)`
- `working_path(name)`
- `public_path(name)`
- `web_project_path(name)`
- `pipeline_check_report(version)`

## Scripts Now Using ProjectPaths

- `scripts/inspect_reading_guide_pipeline.py`
- `scripts/check_reading_guide_pipeline.py`

These scripts keep the R2 semantics:

- EPUB intake is required.
- Identity working artifacts are required.
- Intake reports are required.
- Privacy boundary is required.
- `reviewed-draft` is not required.
- Manual review summary is not required.
- Final public content counts are not required.

## Why Build Logic Is Unchanged

A1 is a path abstraction step only. It does not build public reading-guide data, does not modify the web app, and does not promote project status.

The project is still `draft + intake_ready`; changing build logic before the intermediate `letters_brief` and manual review gates exist would mix unrelated concerns into the path resolver step.

## v0.7-A2 Recommendation

Implement `build_letters_brief.py` next. It should use `ProjectPaths` to:

- read private EPUB extraction artifacts from `private/`
- write a compact working artifact into `working/`
- avoid writing public JSON
- avoid exposing EPUB body text, full chapter text, or long excerpts
- keep the project in draft/intake-ready until review gates are added
